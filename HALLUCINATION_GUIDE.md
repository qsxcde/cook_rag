# RAG 系统防幻觉优化指南

## 🔍 问题分析

根据评估结果，当前系统存在以下幻觉问题：

| 问题 | 示例 | 影响 |
|------|------|------|
| 检索文档为空时仍生成回答 | 白灼虾测试中 `retrieved_docs: []`，但仍生成详细菜谱 | ⭐⭐⭐ 严重 |
| 编造具体数据 | 克数、时间、品牌等不在食谱中的数据 | ⭐⭐⭐ 严重 |
| 虚构引用来源 | 声称"根据食谱 1"但实际没有 | ⭐⭐ 中等 |
| 过度推断 | 添加食谱中没有的内容 | ⭐⭐ 中等 |

---

## ✅ 已实施的防幻觉措施

### 1. **检索前置检查**
所有生成函数在开头添加：
```python
if not context_docs or len(context_docs) == 0:
    return "抱歉，我在当前知识库中没有找到与您问题相关的食谱信息。请尝试其他问题。"
```

### 2. **严格的提示词约束**
所有提示词都包含以下 5 条规则：
```
1. 只回答在「相关食谱信息」中有明确记载的内容
2. 如果信息不足或不确定，必须明确说「抱歉，我在当前食谱中没有找到相关信息」，不要编造
3. 不要声称「根据提供的食谱」却没有实际引用
4. 不要添加食谱中没有的具体数据（如克数、时间、品牌等），除非食谱中明确给出
5. 不要说「根据食谱 1」或「原文」，除非确实有多个明确来源
```

### 3. **评估器修复**
修复了评估器中检索文档的获取方式，确保能正确记录检索结果。

## 🎯 更多防幻觉优化建议

### 方案 A：降低 LLM 温度
```python
# config.py
temperature: float = 0.05  # 从 0.1 降到 0.05
```
- 优点：更保守，减少随机性
- 缺点：回答可能过于机械

### 方案 B：添加源引用标注
在回答中明确标注信息来源：
```
根据《白灼虾》食谱：
- 步骤1：...
- 步骤2：...
```

### 方案 C：添加置信度评分
让 LLM 对每个回答给出置信度：
```
【置信度：高】
根据检索到的 2 个食谱...
```

### 方案 D：双重验证
先生成回答，再让另一个 LLM 检查回答是否基于检索文档。
